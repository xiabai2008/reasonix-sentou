# 报告生成 + 自动修复（CypherFix 模式）

> 借鉴 RedAmon CypherFix / Strix auto-fix：不只是报漏洞，而是直接修好、开好 PR。

---

## Phase 1 — 发现分类与去重（AI Triage）

原始扫描结果（nuclei/fscan/xray）通常有大量重复和误报。先用 AI 做一次分类：

| 步骤 | 操作 | 说明 |
|:-----|:-----|:-----|
| 1. 去重 | 按 CVE/漏洞类型/URL 合并相同发现 | nuclei 对同一端点可能报多次 |
| 2. 误报过滤 | 标注"info"级发现并降权 | info 通常不需要修复 |
| 3. CVSS 评分 | 对每个确定漏洞标注 CVSS 3.1 | 参考 NVD/CVE 数据库 |
| 4. 可利用性排名 | 按"远程可达 + 公开PoC + 无WAF"排序 | 高风险优先修复 |

### 分类输出模板
```json
{
  "finding_id": "F-001",
  "title": "Apache 2.4.49 路径穿越",
  "cve": "CVE-2021-41773",
  "cvss": 7.5,
  "severity": "high",
  "affected_url": "http://target/cgi-bin/",
  "exploitability": "remote_no_auth",
  "poc_available": true,
  "waf_detected": false,
  "verified": true,
  "fix_suggestion": "升级 Apache 到 2.4.51+",
  "auto_fixable": true
}
```

## Phase 2 — 生成结构化报告

### HTML 报告（首选，含交互式 PoC）
```bash
python tools/report_renderers/generate_html.py -i findings.json -o report.html
```

### Word 报告（归档/客户交付）
```bash
python tools/report_renderers/generate_word.py -i findings.json -o report.docx
```

### 报告模板应包含
- 执行摘要（总览、Top 3 风险、修复优先级）
- 漏洞详情（CVSS 向量、PoC 截图、复现步骤）
- 修复建议（代码级 patch、配置修改）
- 合规对照（OWASP Top 10 / PCI DSS / SOC2）

## Phase 3 — CypherFix：自动开修复 PR

> 核心思路：发现→AI 生成修复代码→克隆仓库→应用 patch→开 PR。借鉴 RedAmon CypherFix，但用 Reasonix shell 工具直接操作 git。

### 自动修复类型

| 漏洞类型 | 自动修复方式 | 成功率 |
|:---------|:------------|:------|
| 依赖漏洞（npm/pip CVE） | `npm audit fix` / `pip install --upgrade` | 高 |
| 安全头缺失 | 注入 CSP/HSTS/X-Frame-Options 到 nginx/apache 配置 | 高 |
| 默认口令 | 生成强密码 + 修改配置 | 高 |
| XSS（简单反射型） | 注入 `htmlspecialchars()` / 模板引擎自动转义 | 中 |
| SQL 注入 | 参数化查询改写（需人工 review） | 中 |
| 逻辑漏洞 | 仅生成分析报告，需人工修复 | 低 |

### 自动 PR 工作流
```bash
# 1. AI 分析漏洞，生成修复 patch
# 2. 克隆目标仓库（只读）
git clone <target-repo> /tmp/fix-<project>

# 3. 应用修复
cd /tmp/fix-<project>
# AI 生成的具体修复命令，例如：
# npm audit fix
# 或修改配置文件
# 或注入代码修复

# 4. 提交并开 PR
git checkout -b fix/<cve-id>-<short-desc>
git add -A
git commit -m "fix: CVE-XXXX-XXXXX - <summary>

Auto-generated by Reasonix pentest co-pilot (CypherFix)

Vulnerability: <description>
CVSS: <score>
Verified PoC: <url>

This PR was automatically created by AI analysis of a verified
security finding. Please review the changes before merging."

gh pr create --title "fix: <CVE> - <summary>" --body "..."
```

### ⚠️ 人工审核要求
- 自动生成的修复代码**必须经过人工 review** 才能合并
- SQL 注入/反序列化/逻辑漏洞的自动修复默认不提交，仅生成 patch 文件供人工审查
- 生产环境修复需先在 staging 验证
