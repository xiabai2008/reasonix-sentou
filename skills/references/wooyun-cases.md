## 三、越权测试协议（IDOR + 垂直越权）

### 水平越权/IDOR — 破晓在发现带用户数据的 API 时必须执行

```
对于每个返回用户特定数据的 API 端点：

1. 识别资源标识符
   - URL 路径：/api/users/{id}/orders
   - 查询参数：/api/orders?user_id=123
   - POST 请求体：{"user_id": 123}
   - Cookie/头：X-User-Id: 123

2. 测试每项 CRUD 操作
   - [ ] 读取：A 能否查看 B 的资源？（最常见）
   - [ ] 更新：A 能否修改 B 的资源？
   - [ ] 删除：A 能否删除 B 的资源？

3. 测试标识符操纵
   - [ ] 直接 ID 替换：123 → 124
   - [ ] ID 枚举：遍历 ID 范围
   - [ ] 参数污染：?uid=123&uid=456
   - [ ] 数组注入：uid[]=123
   - [ ] JSON 嵌套：{"user": {"id": 456}}
   - [ ] 负数/零 ID：id=0、id=-1

4. 额外技巧
   - [ ] 改变 HTTP 方法：GET→POST→PUT→DELETE
   - [ ] 不带任何 Cookie/Token 访问管理员端点
   - [ ] X-Forwarded-For: 127.0.0.1 绕过 IP 限制
   - [ ] URL 编码：/%61dmin 绕过路径保护
```

### 任意操作/任意X（WooYun 特有分类）

| 子类别 | 测试要点 | 示例 |
|--------|---------|------|
| 任意账号访问 | 不需凭证以任何用户身份登录/操作 | 修改他人手机号→重置密码→登录 |
| 任意修改 | 修改任何记录（资料、配置、内容） | 修改他人订单收货地址 |
| 任意查看 | 查看批量导出、管理员视图 | 遍历他人身份证件/驾驶证 |
| 任意删除 | 删除任何记录无需所有权 | 删除他人发票记录 |
| 任意操作 | 执行特权操作（批准、发布） | 普通用户审批管理员操作 |

## 四、SQL 注入攻击模式分布

```
注入: 42次 → 直接用于数据提取或getshell
泄露: 4次  → UNION查询泄露敏感信息
遍历: 1次  → 盲注遍历数据库
getshell: 1次 → INTO OUTFILE / UDF提权
```

### WooYun 真实 SQL 注入案例特征
- **CMS 类**：Dswjcms、KingCms、谷秋精品课程、律师建站系统 → 同文件多参数注入
- **行业类**：电信商企平台、图书检索系统、NatShell认证计费 → 特定行业系统通用漏洞
- **权限等级**：大量案例达 DBA/root 权限，可直接 `INTO OUTFILE` 写 webshell

### 破晓分析 SQL 注入时的检查清单
```
1. 确认注入点参数类型（数字/字符/搜索）
2. 确认数据库类型（MySQL/MSSQL/Oracle/PostgreSQL）
3. 确认权限等级（DBA / 普通用户）
4. 确认是否可 UNION 查询（判断列数）
5. 确认是否可 OUT FILE（MySQL DBA）
6. 确认是否可 xp_cmdshell（MSSQL SA）
7. 确认 WAF 类型及绕过方式
```

## 七、防御模式（供破晓生成修复建议时参考）

### 越权防御
- **默认拒绝**：公开端点加白名单，其他需要认证
- **所有权验证**：`resource.owner_id == current_user.id`
- **不可预测 ID**：UUID v4，非顺序整数
- **RBAC/ABAC**：基于角色或属性的访问控制

### 逻辑漏洞防御
- **服务端状态机**：不允许客户端控制流程跳转
- **服务端金额校验**：所有金额以服务端计算为准
- **幂等性设计**：支付/订单操作加唯一 Token 防重放

### 弱口令防御
- **密码复杂度策略**：最小8位+大小写+数字+特殊字符
- **登录失败锁定**：5次失败锁定15分钟
- **多因素认证**：短信/邮箱/TOTP 双因子

## 使用方式

破晓分析漏洞时，对照本文件：
1. 确认漏洞类型 → 查"漏洞频率全景"看属于高频/高危类型
2. 生成测试步骤 → 用"越权测试协议"或"SQL注入检查清单"
3. 写报告 → 引用"真实案例速查"中相似案例佐证
4. 给修复建议 → 用"防御模式"中的最佳实践
